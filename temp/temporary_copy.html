US01_Login Function

Section Name

Section Description

User Story

As an end-user, I want to be able to log into the website using my credentials to access my personalized content and settings.

Pre-condition

User must have an active account with a username and password already set up.

Description with User Workflow

1. User navigates to the login page of the website.
2. User is presented with fields to enter username and password.
3. User enters their username and password and clicks the 'Log In' button.
4. System validates the credentials against the stored user database.
5. If credentials are correct, user is redirected to their personal dashboard.
6. If credentials are incorrect, an error message is displayed, and the user can attempt to log in again or reset their password.

Post-condition

User is successfully logged in to the website and directed to their personal dashboard, where they can access personalized content and settings.

Acceptance Criteria

1. Login page is accessible and loads without error.
2. User can enter username and password.
3. System correctly validates credentials.
4. Upon successful login, user is redirected to their dashboard.
5. Unsuccessful login attempts result in an appropriate error message with an option to reset password.
6. User can log out from the dashboard.

Edge Case

1. User forgets their password and requires a password reset.
2. User attempts to log in with an incorrect username or password multiple times and is temporarily locked out.
3. User has disabled JavaScript, and the login functionality degrades gracefully.
4. User's account is inactive or suspended, preventing login.
